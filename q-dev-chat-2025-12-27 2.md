Based on the conversation history, you need to cross-compile the Pulse project from Windows to Linux. Here's what you need:

## Required Tools for Cross-Compilation

### 1. **Go with CGO Support**
You already have Go 1.25.5 installed, but for CGO cross-compilation you need:

**Install MinGW-w64 (GCC for Windows):**
```powershell
# Using Chocolatey
choco install mingw

# Or download from: https://www.mingw-w64.org/downloads/
```

### 2. **Cross-Compilation Setup**

After installing MinGW, set up for Linux cross-compilation:

```powershell
# Set environment variables
$env:GOOS = "linux"
$env:GOARCH = "amd64"
$env:CGO_ENABLED = "1"
$env:CC = "x86_64-w64-mingw32-gcc"  # For cross-compile

# Build
go build -o pulse-server.linux ./cmd/pulse
```

## Easier Alternative: Build Directly in LXC

Since CGO cross-compilation on Windows is complex, **build directly in your LXC container**:

```bash
# SSH to Proxmox
ssh root@<proxmox-ip>

# Enter LXC container
pct enter <container-id>

# Install Go if not present
apt update && apt install golang-go git

# Clone/update source
cd /opt && git clone <your-repo> pulse-src
cd pulse-src

# Build with CGO
CGO_ENABLED=1 go build -o /usr/local/bin/pulse ./cmd/pulse

# Restart service
systemctl restart pulse
```

To enable CGO for cross-compilation from Windows to Linux, you need a proper Linux cross-compiler toolchain. MinGW is for Windows targets, not Linux.

## Install Linux Cross-Compiler on Windows

**Option 1: Use WSL2 (Recommended)**
```powershell

wsl
```

**Option 2: Install Zig (Cross-compile with CGO)**
```powershell
# Install Zig
choco install zig

# Use Zig as C compiler for cross-compilation
$env:CGO_ENABLED = "1"
$env:GOOS = "linux"
$env:GOARCH = "amd64"
$env:CC = "zig cc -target x86_64-linux-gnu"
$env:CXX = "zig c++ -target x86_64-linux-gnu"
go build -tags=prod -o pulse-linux ./cmd/pulse
```
