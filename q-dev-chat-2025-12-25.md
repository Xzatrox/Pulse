## Implementation Strategy for Detecting Installed Apps Using osquery

Here's the minimal approach:

### **Strategy Overview**

1. **Execute osquery locally** - Run `osqueryi` or `osqueryd` binary on the host
2. **Query installed packages** - Use osquery's built-in tables
3. **Parse JSON results** - osquery outputs JSON
4. **Report to Pulse server** - Send via existing agent HTTP client

### **Key osquery Tables for Installed Apps**

- **Linux**: `deb_packages`, `rpm_packages`, `portage_packages`, `npm_packages`, `python_packages`
- **Windows**: `programs`, `chocolatey_packages`
- **macOS**: `apps`, `homebrew_packages`
- **Cross-platform**: `docker_images`, `docker_containers`

### **Minimal Implementation**

```go
// internal/osqueryagent/collector.go
package osqueryagent

import (
    "encoding/json"
    "os/exec"
)

type Package struct {
    Name    string `json:"name"`
    Version string `json:"version"`
    Source  string `json:"source"`
}

func (a *Agent) collectInstalledApps() ([]Package, error) {
    query := "SELECT name, version, 'deb' as source FROM deb_packages UNION SELECT name, version, 'rpm' as source FROM rpm_packages;"
    
    cmd := exec.Command("osqueryi", "--json", query)
    output, err := cmd.Output()
    if err != nil {
        return nil, err
    }
    
    var packages []Package
    json.Unmarshal(output, &packages)
    return packages, nil
}
```

### **Integration Points**

1. **Check osquery binary exists** - `exec.LookPath("osqueryi")`
2. **Run queries periodically** - In `Run()` ticker loop
3. **Send to Pulse API** - POST to `/api/agents/{id}/packages`
4. **Cache results** - Only send changes to reduce bandwidth

### **Advantages of This Approach**

- No osquery daemon required
- Uses existing osquery installation
- Minimal dependencies (just exec)
- Works cross-platform
- Leverages osctrl types for future expansion

### **Alternative: Embedded osquery**

If you want deeper integration, you could:
- Embed osquery as a library (complex)
- Run osqueryd and use osctrl's TLS endpoint (heavier)
- Use osquery extensions API (advanced)

**Recommendation**: Start with the exec approach - it's minimal, reliable, and matches your requirement for "ABSOLUTE MINIMAL code."
